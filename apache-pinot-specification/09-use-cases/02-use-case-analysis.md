## 2. 适用场景详解

### 2.1 用户面向实时分析

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        场景：用户面向实时分析                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  业务需求：                                                              │
│  ─────────────────                                                      │
│  ├── 用户直接查询分析结果                                                │
│  ├── 查询延迟必须 < 1 秒                                                 │
│  ├── 高并发查询（数千 QPS）                                              │
│  └── 数据实时更新                                                        │
│                                                                          │
│  典型案例：                                                              │
│  ─────────────────                                                      │
│  ├── LinkedIn：用户查看个人档案访问统计                                  │
│  ├── Uber：司机查看实时收入统计                                          │
│  ├── Slack：团队消息统计分析                                             │
│  ├── 电商平台：卖家实时销售数据                                          │
│                                                                          │
│  为什么 Pinot 适合：                                                     │
│  ─────────────────                                                      │
│  ├── 亚秒级延迟满足用户体验                                              │
│  ├── 高并发支持数千用户同时查询                                          │
│  ├── 实时摄入保证数据新鲜度                                              │
│  └── 索引优化减少计算开销                                                │
│                                                                          │
│  数据规模示例：                                                          │
│  ─────────────────                                                      │
│  ├── 日活用户：1000 万                                                   │
│  ├── 日事件量：10 亿                                                     │
│  ├── 查询 QPS：50000                                                     │
│  ├── 查询延迟：P99 < 100ms                                               │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

**示例查询：**

```sql
-- 用户查看个人统计
SELECT 
  COUNT(*) AS total_views,
  COUNT(DISTINCT viewer_id) AS unique_viewers,
  SUM(engagement_time) AS total_engagement
FROM profile_views
WHERE user_id = 'user123'
  AND timestamp > now() - 86400000;

-- 实时收入统计
SELECT 
  DATE_TRUNC('hour', TO_TIMESTAMP(timestamp / 1000)) AS hour,
  SUM(revenue) AS hourly_revenue,
  COUNT(*) AS trips
FROM driver_revenue
WHERE driver_id = 'driver456'
  AND timestamp > now() - 3600000
GROUP BY hour;
```

**示例查询：**

```sql
-- 实时错误率监控
SELECT 
  service_name,
  COUNT(*) AS total_requests,
  SUM(CASE WHEN status_code >= 400 THEN 1 ELSE 0 END) AS error_count,
  SUM(CASE WHEN status_code >= 400 THEN 1 ELSE 0 END) * 100.0 / COUNT(*) AS error_rate
FROM request_logs
WHERE timestamp > now() - 60000
GROUP BY service_name
HAVING error_rate > 5;

-- 异常流量检测
SELECT 
  source_ip,
  COUNT(*) AS request_count,
  COUNT(DISTINCT target_url) AS unique_urls
FROM network_logs
WHERE timestamp > now() - 60000
GROUP BY source_ip
HAVING request_count > 1000
ORDER BY request_count DESC;
```

**示例查询：**

```sql
-- 用户实时兴趣特征
SELECT 
  category,
  COUNT(*) AS view_count,
  SUM(CASE WHEN event_type = 'purchase' THEN 1 ELSE 0 END) AS purchase_count,
  AVG(session_duration) AS avg_duration
FROM user_events
WHERE user_id = 'user123'
  AND timestamp > now() - 86400000
GROUP BY category
ORDER BY purchase_count DESC, view_count DESC;

-- 热门商品实时排行
SELECT 
  product_id,
  COUNT(*) AS view_count,
  COUNT(DISTINCT user_id) AS unique_viewers,
  SUM(CASE WHEN event_type = 'purchase' THEN 1 ELSE 0 END) AS purchase_count
FROM product_events
WHERE timestamp > now() - 3600000
GROUP BY product_id
ORDER BY purchase_count DESC
LIMIT 100;
```

## 4. 不适用场景详解

### 4.1 事务型应用（OLTP）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        不适用：事务型应用                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  为什么不适合：                                                          │
│  ─────────────────                                                      │
│  ├── Pinot 不支持 ACID 事务                                              │
│  ├── 不支持行级锁                                                        │
│  ├── 不支持多表事务                                                      │
│  ├── 写入延迟较高                                                        │
│                                                                          │
│  典型场景：                                                              │
│  ─────────────────                                                      │
│  ├── 订单管理系统                                                        │
│  ├── 用户账户管理                                                        │
│  ├── 库存管理系统                                                        │
│  ├── 支付交易系统                                                        │
│                                                                          │
│  推荐替代方案：                                                          │
│  ─────────────────                                                      │
│  ├── MySQL / PostgreSQL（关系型数据库）                                  │
│  ├── MongoDB（文档数据库）                                               │
│  ├── CockroachDB（分布式事务数据库）                                     │
│  ├── TiDB（NewSQL 数据库）                                               │
│                                                                          │
│  正确做法：                                                              │
│  ─────────────────                                                      │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐                          │
│  │ 事务系统  │───>│  数据同步  │───>│  Pinot   │                          │
│  │ (MySQL)  │    │  (CDC)    │    │  分析     │                          │
│  └──────────┘    └──────────┘    └──────────┘                          │
│                                                                          │
│  MySQL 处理事务，Pinot 处理分析                                          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 8.8 选型建议

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        选型建议                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  选择 Pinot 的场景：                                                     │
│  ─────────────────                                                      │
│  ├── 查询延迟要求 < 100ms                                                │
│  ├── 高并发查询（> 1000 QPS）                                            │
│  ├── 用户面向实时分析                                                    │
│  ├── 实时监控仪表盘                                                      │
│  ├── 查询模式固定                                                        │
│  ├── 已有 Kafka 流式数据管道                                             │
│                                                                          │
│  选择 Doris 的场景：                                                     │
│  ─────────────────                                                      │
│  ├── 需要完整 SQL 支持                                                   │
│  ├── 复杂多表关联查询                                                    │
│  ├── 数据仓库建设                                                        │
│  ├── BI 工具集成                                                         │
│  ├── 需要数据湖联邦查询                                                  │
│  ├── 运维简单优先                                                        │
│  ├── MySQL 协议兼容需求                                                  │
│                                                                          │
│  组合方案：                                                              │
│  ─────────────────                                                      │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐                          │
│  │ 实时数据  │───>│  Pinot   │───>│ 用户面向  │                          │
│  │ Kafka    │    │ 实时分析  │    │ 仪表盘    │                          │
│  └──────────┘    └──────────┘    └──────────┘                          │
│       │                                                              │
│       │              ┌──────────┐    ┌──────────┐                      │
│       └─────────────>│  Doris   │───>│ 数据仓库  │                      │
│                      │ 批量分析  │    │ BI 报表   │                      │
│                      └──────────┘    └──────────┘                      │
│                                                                          │
│  Pinot 处理实时低延迟查询，Doris 处理复杂分析                             │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```
