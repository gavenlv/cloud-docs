# Pinot 索引策略

## 概述

本文档介绍 Apache Pinot 的索引类型和选择策略，帮助优化查询性能。

---

## 1. 索引类型概览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Pinot 索引类型                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  索引类型              │ 适用场景              │ 存储开销 │ 查询加速    │
│  ──────────────────────┼───────────────────────┼──────────┼─────────────│
│  Forward Index         │ 默认，所有列          │ 低       │ 基础        │
│  Inverted Index        │ 等值过滤、IN 查询     │ 中       │ 高          │
│  Sorted Index          │ 范围查询、排序        │ 低       │ 很高        │
│  Range Index           │ 范围过滤              │ 中       │ 高          │
│  Bloom Filter          │ 存在性检查            │ 低       │ 中          │
│  Star-Tree Index       │ 聚合查询              │ 高       │ 极高        │
│  JSON Index            │ JSON 数据查询         │ 中       │ 高          │
│  Text Index            │ 全文搜索              │ 高       │ 高          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## 2. 索引详解

### 2.1 Forward Index

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Forward Index（前向索引）                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  特点：                                                                  │
│  ─────────────────                                                      │
│  ├── 默认索引，所有列都有                                                 │
│  ├── 存储列的原始值                                                       │
│  ├── 支持快速顺序扫描                                                     │
│  └── 存储格式：字典编码 + 文档 ID 列表                                    │
│                                                                          │
│  存储格式：                                                              │
│  ─────────────────                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  原始数据：                                                      │   │
│  │  ┌─────────┬─────────┐                                         │   │
│  │  │ DocId   │ Country │                                         │   │
│  │  ├─────────┼─────────┤                                         │   │
│  │  │ 0       │ US      │                                         │   │
│  │  │ 1       │ UK      │                                         │   │
│  │  │ 2       │ US      │                                         │   │
│  │  │ 3       │ CA      │                                         │   │
│  │  └─────────┴─────────┘                                         │   │
│  │                                                                  │   │
│  │  Forward Index 存储：                                            │   │
│  │  ┌─────────────────┐    ┌─────────────────┐                    │   │
│  │  │  字典            │    │  文档 ID 列表    │                    │   │
│  │  │  ┌─────────┐    │    │  ┌─────────┐    │                    │   │
│  │  │  │ 0: US   │    │    │  │ 0: 0    │    │                    │   │
│  │  │  │ 1: UK   │    │    │  │ 1: 1    │    │                    │   │
│  │  │  │ 2: CA   │    │    │  │ 2: 0    │    │                    │   │
│  │  │  └─────────┘    │    │  │ 3: 2    │    │                    │   │
│  │  └─────────────────┘    │  └─────────┘    │                    │   │
│  │                         └─────────────────┘                    │   │
│  │                                                                  │   │
│  │  查询：SELECT country FROM table WHERE docId = 2               │   │
│  │  结果：查找 docId=2 的文档 ID 列表 -> 字典值 0 -> "US"          │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 Inverted Index

```json
{
  "tableIndexConfig": {
    "invertedIndexColumns": ["user_id", "country", "event_type"]
  }
}
```

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Inverted Index（倒排索引）                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  特点：                                                                  │
│  ─────────────────                                                      │
│  ├── 适用：等值过滤、IN 查询                                              │
│  ├── 结构：值 -> 文档 ID 列表                                             │
│  ├── 查询：WHERE country = 'US'                                          │
│  └── 开销：中等（额外存储映射关系）                                       │
│                                                                          │
│  存储格式：                                                              │
│  ─────────────────                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Inverted Index：                                                │   │
│  │  ┌─────────┬─────────────────┐                                  │   │
│  │  │ Value   │ DocId List      │                                  │   │
│  │  ├─────────┼─────────────────┤                                  │   │
│  │  │ US      │ [0, 2, 5, 8]    │                                  │   │
│  │  │ UK      │ [1, 4, 7]       │                                  │   │
│  │  │ CA      │ [3, 6, 9]       │                                  │   │
│  │  └─────────┴─────────────────┘                                  │   │
│  │                                                                  │   │
│  │  查询：SELECT * FROM table WHERE country = 'US'                 │   │
│  │  结果：直接读取 DocId List [0, 2, 5, 8]，无需扫描所有数据       │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.3 Sorted Index

```json
{
  "tableIndexConfig": {
    "sortedColumn": ["timestamp"]
  }
}
```

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Sorted Index（排序索引）                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  特点：                                                                  │
│  ─────────────────                                                      │
│  ├── 适用：范围查询、排序                                                 │
│  ├── 要求：数据按该列排序                                                 │
│  ├── 优势：范围查询只需扫描部分数据                                       │
│  └── 开销：低（利用数据有序性）                                           │
│                                                                          │
│  存储格式：                                                              │
│  ─────────────────                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  排序数据：                                                      │   │
│  │  ┌─────────┬─────────────────┐                                  │   │
│  │  │ DocId   │ Timestamp       │                                  │   │
│  │  ├─────────┼─────────────────┤                                  │   │
│  │  │ 0       │ 1704067200000   │                                  │   │
│  │  │ 1       │ 1704067260000   │                                  │   │
│  │  │ 2       │ 1704067320000   │                                  │   │
│  │  │ 3       │ 1704067380000   │                                  │   │
│  │  │ 4       │ 1704067440000   │                                  │   │
│  │  └─────────┴─────────────────┘                                  │   │
│  │                                                                  │   │
│  │  查询：SELECT * FROM table                                      │   │
│  │         WHERE timestamp > 1704067300000                         │   │
│  │  结果：使用二分查找定位到 DocId=2，只扫描 [2, 3, 4]             │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.4 Star-Tree Index

```json
{
  "tableIndexConfig": {
    "starTreeIndexConfigs": [
      {
        "dimensionsSplitOrder": ["country", "city"],
        "functionColumnPairs": ["SUM__revenue", "COUNT__*"],
        "maxLeafRecords": 10000
      }
    ]
  }
}
```

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Star-Tree Index（星型树索引）                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  特点：                                                                  │
│  ─────────────────                                                      │
│  ├── 适用：聚合查询加速                                                   │
│  ├── 原理：预计算多维聚合                                                 │
│  ├── 优势：聚合查询性能提升 10-100 倍                                     │
│  └── 开销：高（预计算存储）                                               │
│                                                                          │
│  结构：                                                                  │
│  ─────────────────                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                    Star-Tree 结构                                │   │
│  │                                                                  │   │
│  │                         * (ALL)                                  │   │
│  │                         │                                        │   │
│  │            ┌────────────┼────────────┐                          │   │
│  │            ▼            ▼            ▼                          │   │
│  │          US            UK         ALL                            │   │
│  │            │            │                                        │   │
│  │      ┌─────┴─────┐     │                                        │   │
│  │      ▼           ▼     ▼                                        │   │
│  │     NY          LA   London                                      │   │
│  │      │           │     │                                        │   │
│  │   ┌──┴──┐     ┌─┴─┐  ┌─┴─┐                                     │   │
│  │   ▼     ▼     ▼   ▼  ▼   ▼                                     │   │
│  │  sum  count  sum count sum count                               │   │
│  │  220   2     200  1   150  1                                    │   │
│  │                                                                  │   │
│  │  查询：SELECT SUM(revenue) FROM table WHERE country='US'        │   │
│  │  结果：直接从 Star-Tree 节点读取 = 420                            │   │
│  │  加速：无需扫描原始数据                                           │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3. 索引选择策略

### 3.1 决策树

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        索引选择决策树                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  开始                                                                     │
│   │                                                                      │
│   ▼                                                                      │
│  列是否用于过滤？                                                         │
│   │                                                                      │
│   ├── 是 ──> 过滤类型？                                                  │
│   │           │                                                          │
│   │           ├── 等值/IN ──> 基数？                                     │
│   │           │               │                                          │
│   │           │               ├── 低基数（<10K）──> Inverted Index       │
│   │           │               └── 高基数 ──> Bloom Filter                │
│   │           │                                                          │
│   │           └── 范围 ──> 数据是否有序？                                │
│   │                       │                                              │
│   │                       ├── 是 ──> Sorted Index                        │
│   │                       └── 否 ──> Range Index                         │
│   │                                                                      │
│   └── 否 ──> 列是否用于聚合？                                            │
│               │                                                          │
│               ├── 是 ──> 聚合频率？                                      │
│               │           │                                              │
│               │           ├── 高频 ──> Star-Tree Index                   │
│               │           └── 低频 ──> Forward Index                     │
│               │                                                          │
│               └── 否 ──> Forward Index（默认）                           │
│                                                                          │
│  时间戳列特殊处理：                                                        │
│  ─────────────────                                                      │
│  ├── 总是设置为 sortedColumn                                             │
│  ├── 添加 Range Index                                                    │
│  └── 考虑时间分区策略                                                    │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 推荐配置

```json
{
  "tableIndexConfig": {
    "loadMode": "MMAP",
    "invertedIndexColumns": [
      "user_id",
      "country",
      "event_type",
      "device_type"
    ],
    "sortedColumn": ["timestamp"],
    "rangeIndexColumns": ["timestamp", "revenue"],
    "bloomFilterColumns": ["user_id", "session_id"],
    "starTreeIndexConfigs": [
      {
        "dimensionsSplitOrder": ["country", "event_type", "device_type"],
        "functionColumnPairs": [
          "SUM__revenue",
          "COUNT__*",
          "AVG__revenue"
        ],
        "maxLeafRecords": 10000
      }
    ],
    "jsonIndexConfigs": [
      {
        "columnName": "properties",
        "maxLevels": 2
      }
    ],
    "enableDefaultStarTree": false,
    "enableDynamicStarTreeCreation": true
  }
}
```

## 4. 性能测试

### 4.1 基准测试

```bash
# 生成测试数据
bin/pinot-admin.sh GenerateData \
  -numRecords 10000000 \
  -numFiles 10 \
  -schemaFile schema.json \
  -outDir /tmp/test-data

# 测试查询性能
curl -X POST http://pinot-broker:8099/query/sql \
  -H "Content-Type: application/json" \
  -d '{
    "sql": "SELECT country, SUM(revenue) FROM test_table WHERE timestamp > now() - 86400000 GROUP BY country",
    "trace": true
  }'
```
