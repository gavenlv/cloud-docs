# Apache Pinot 原理与架构

## 概述

Apache Pinot 是一个开源的分布式 OLAP（Online Analytical Processing）数据库，专为实时分析而设计。它能够在亚秒级延迟内处理大规模数据，支持低延迟的实时查询和高吞吐量的批量查询。

## 4. 索引机制

### 4.1 索引类型对比

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Pinot 索引类型对比                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  索引类型              │ 适用场景              │ 存储开销 │ 查询加速    │
│  ──────────────────────┼───────────────────────┼──────────┼─────────────│
│  Forward Index         │ 默认，所有列          │ 低       │ 基础        │
│  Inverted Index        │ 等值过滤、IN 查询     │ 中       │ 高          │
│  Sorted Index          │ 范围查询、排序        │ 低       │ 很高        │
│  Range Index           │ 范围过滤              │ 中       │ 高          │
│  Bloom Filter          │ 存在性检查            │ 低       │ 中          │
│  Star-Tree Index       │ 聚合查询              │ 高       │ 极高        │
│  JSON Index            │ JSON 数据查询         │ 中       │ 高          │
│  Text Index            │ 全文搜索              │ 高       │ 高          │
│                                                                          │
│  索引选择建议：                                                          │
│  ─────────────────────────────                                          │
│  ├── 主键/ID 列：Inverted Index + Bloom Filter                          │
│  ├── 时间戳列：Sorted Index                                              │
│  ├── 维度列（低基数）：Inverted Index                                    │
│  ├── 维度列（高基数）：Bloom Filter                                      │
│  ├── 度量列：Forward Index（无需额外索引）                               │
│  └── 频繁聚合查询：Star-Tree Index                                       │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 Star-Tree Index 详解

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Star-Tree Index 原理                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  问题：聚合查询需要扫描大量数据                                          │
│                                                                          │
│  解决方案：预计算聚合结果                                                │
│                                                                          │
│  原始数据：                                                              │
│  ┌────────────┬──────────┬──────────┬──────────┐                      │
│  │  timestamp │  country │  city    │  revenue │                      │
│  ├────────────┼──────────┼──────────┼──────────┤                      │
│  │ 2024-01-01 │  US      │  NY      │  100     │                      │
│  │ 2024-01-01 │  US      │  LA      │  200     │                      │
│  │ 2024-01-01 │  UK      │  London  │  150     │                      │
│  │ 2024-01-02 │  US      │  NY      │  120     │                      │
│  │ ...        │  ...     │  ...     │  ...     │                      │
│  └────────────┴──────────┴──────────┴──────────┘                      │
│                                                                          │
│  Star-Tree 预计算：                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                  │   │
│  │                    Star-Tree 结构                                │   │
│  │                                                                  │   │
│  │                         * (ALL)                                  │   │
│  │                         │                                        │   │
│  │            ┌────────────┼────────────┐                          │   │
│  │            ▼            ▼            ▼                          │   │
│  │          US            UK         ALL                            │   │
│  │            │            │                                        │   │
│  │      ┌─────┴─────┐     │                                        │   │
│  │      ▼           ▼     ▼                                        │   │
│  │     NY          LA   London                                      │   │
│  │      │           │     │                                        │   │
│  │   ┌──┴──┐     ┌─┴─┐  ┌─┴─┐                                     │   │
│  │   ▼     ▼     ▼   ▼  ▼   ▼                                     │   │
│  │  sum  count  sum count sum count                               │   │
│  │  220   2     200  1   150  1                                    │   │
│  │                                                                  │   │
│  │  查询：SELECT SUM(revenue) FROM table WHERE country='US'        │   │
│  │  结果：直接从 Star-Tree 节点读取 = 420                            │   │
│  │  加速：无需扫描原始数据                                           │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
│  配置示例：                                                              │
│  ─────────────────                                                      │
│  {                                                                      │
│    "starTreeIndexConfigs": [{                                           │
│      "dimensionsSplitOrder": ["country", "city"],                      │
│      "functionColumnPairs": ["SUM__revenue", "COUNT__*"]               │
│    }]                                                                   │
│  }                                                                      │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 5. 查询处理流程

### 5.1 查询生命周期

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Pinot 查询生命周期                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  客户端                                                                  │
│    │                                                                     │
│    │  1. 发送查询                                                        │
│    │     SELECT SUM(revenue) FROM orders                                 │
│    │     WHERE country = 'US' AND timestamp > '2024-01-01'               │
│    ▼                                                                     │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                        Broker                                    │   │
│  │                                                                  │   │
│  │  2. 查询解析                                                      │   │
│  │     ├── 解析 SQL                                                  │   │
│  │     ├── 验证表和列存在                                            │   │
│  │     └── 生成查询计划                                              │   │
│  │                                                                  │   │
│  │  3. 路由决策                                                      │   │
│  │     ├── 确定时间范围                                              │   │
│  │     ├── 筛选相关 Segment                                          │   │
│  │     └── 选择目标 Server                                           │   │
│  │                                                                  │   │
│  │  4. 分发查询                                                      │   │
│  │     ├── 拆分查询为子查询                                          │   │
│  │     ├── 发送到多个 Server                                         │   │
│  │     └── 并行执行                                                  │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│    │                                                                     │
│    │  5. 并行查询                                                        │
│    ▼                                                                     │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                        Server                                    │   │
│  │                                                                  │   │
│  │  6. Segment 筛选                                                  │   │
│  │     ├── 根据时间范围筛选 Segment                                  │   │
│  │     ├── 应用过滤条件                                              │   │
│  │     └── 使用索引加速                                              │   │
│  │                                                                  │   │
│  │  7. 数据扫描                                                      │   │
│  │     ├── 读取 Forward Index                                        │   │
│  │     ├── 应用谓词过滤                                              │   │
│  │     └── 计算聚合结果                                              │   │
│  │                                                                  │   │
│  │  8. 返回结果                                                      │   │
│  │     ├── 序列化结果                                                │   │
│  │     └── 发送给 Broker                                             │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│    │                                                                     │
│    │  9. 结果聚合                                                        │
│    ▼                                                                     │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                        Broker                                    │   │
│  │                                                                  │   │
│  │  10. 合并结果                                                     │   │
│  │      ├── 收集所有 Server 结果                                     │   │
│  │      ├── 合并聚合值                                               │   │
│  │      ├── 排序（如需要）                                           │   │
│  │      └── 限制返回行数                                             │   │
│  │                                                                  │   │
│  │  11. 返回客户端                                                   │   │
│  │      ├── 序列化最终结果                                           │   │
│  │      └── 发送给客户端                                             │   │
│  │                                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│    │                                                                     │
│    │  12. 接收结果                                                       │
│    ▼                                                                     │
│  客户端                                                                  │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```
