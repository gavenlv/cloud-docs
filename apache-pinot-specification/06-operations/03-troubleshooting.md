# Pinot 故障排查

## 概述

本文档介绍 Apache Pinot 常见问题的排查方法和解决方案。

---

## 1. 查询问题

### 1.1 查询超时

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        查询超时排查                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  症状：查询返回超时错误                                                  │
│                                                                          │
│  排查步骤：                                                              │
│  ─────────────────                                                      │
│  1. 检查查询复杂度                                                       │
│     ├── 是否扫描了大量数据？                                             │
│     ├── 是否缺少索引？                                                   │
│     └── 是否有大量 JOIN？                                                │
│                                                                          │
│  2. 检查集群负载                                                         │
│     ├── Broker CPU 使用率                                                │
│     ├── Server CPU 使用率                                                │
│     └── 网络带宽                                                         │
│                                                                          │
│  3. 检查 Segment 状态                                                    │
│     ├── Segment 是否已加载？                                             │
│     ├── Segment 大小是否正常？                                           │
│     └── 索引是否已构建？                                                 │
│                                                                          │
│  解决方案：                                                              │
│  ─────────────────                                                      │
│  ├── 增加查询超时时间                                                    │
│  ├── 添加合适的索引                                                      │
│  ├── 优化查询条件（时间范围过滤）                                        │
│  ├── 扩容 Broker/Server                                                  │
│  └── 使用 Star-Tree 索引                                                 │
│                                                                          │
│  查询示例：                                                              │
│  ─────────────────                                                      │
│  -- 增加超时                                                             │
│  SELECT /*+ timeoutMs(30000) */ * FROM orders;                          │
│                                                                          │
│  -- 优化查询                                                             │
│  SELECT * FROM orders                                                    │
│  WHERE timestamp > now() - 3600000  -- 添加时间过滤                      │
│    AND user_id = 'user123';  -- 使用索引列                               │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 查询结果不正确

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        查询结果不正确排查                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  症状：查询结果与预期不符                                                │
│                                                                          │
│  排查步骤：                                                              │
│  ─────────────────                                                      │
│  1. 检查数据摄入                                                         │
│     ├── 数据是否已成功摄入？                                             │
│     ├── 时间戳是否正确？                                                 │
│     └── 数据转换是否正确？                                               │
│                                                                          │
│  2. 检查 Schema                                                          │
│     ├── 字段类型是否正确？                                               │
│     ├── 时间字段格式是否正确？                                           │
│     └── 是否有字段名拼写错误？                                           │
│                                                                          │
│  3. 检查表配置                                                           │
│     ├── 时间列配置是否正确？                                             │
│     ├── 保留策略是否过期了数据？                                         │
│     └── 副本是否一致？                                                   │
│                                                                          │
│  解决方案：                                                              │
│  ─────────────────                                                      │
│  ├── 检查 Segment 元数据                                                 │
│  ├── 重新摄入数据                                                        │
│  ├── 修复 Schema                                                         │
│  └── 触发 Segment 重新加载                                               │
│                                                                          │
│  诊断命令：                                                              │
│  ─────────────────                                                      │
│  -- 查看 Segment 信息                                                    │
│  curl http://pinot-controller:9000/tables/{tableName}/segments           │
│                                                                          │
│  -- 查看 Segment 元数据                                                  │
│  curl http://pinot-server:8097/segments/{tableName}/{segmentName}/metadata
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 2. 摄入问题

### 2.1 实时摄入延迟

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        实时摄入延迟排查                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  症状：数据摄入延迟高，查询不到最新数据                                  │
│                                                                          │
│  排查步骤：                                                              │
│  ─────────────────                                                      │
│  1. 检查 Kafka 消费                                                      │
│     ├── 消费者组是否活跃？                                               │
│     ├── 分区是否有延迟？                                                 │
│     └── 消费者是否卡住？                                                 │
│                                                                          │
│  2. 检查 Server 状态                                                     │
│     ├── Server 是否健康？                                                │
│     ├── 内存是否足够？                                                   │
│     └── CPU 是否饱和？                                                   │
│                                                                          │
│  3. 检查 Segment 构建                                                    │
│     ├── 构建时间是否过长？                                               │
│     ├── 提交是否失败？                                                   │
│     └── 是否有构建错误？                                                 │
│                                                                          │
│  解决方案：                                                              │
│  ─────────────────                                                      │
│  ├── 增加 Server 资源                                                    │
│  ├── 调整 Segment 刷新阈值                                               │
│  ├── 检查 Kafka 集群健康                                                 │
│  └── 重启消费者                                                          │
│                                                                          │
│  诊断命令：                                                              │
│  ─────────────────                                                      │
│  -- 查看消费延迟                                                         │
│  curl http://pinot-server:8097/debug/consumers                           │
│                                                                          │
│  -- 查看 Kafka 消费者组                                                  │
│  kafka-consumer-groups.sh --bootstrap-server kafka:9092 --describe --group pinot-user-events
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 批量摄入失败

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        批量摄入失败排查                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  症状：批量摄入作业失败                                                  │
│                                                                          │
│  排查步骤：                                                              │
│  ─────────────────                                                      │
│  1. 检查数据源                                                           │
│     ├── 数据文件是否存在？                                               │
│     ├── 数据格式是否正确？                                               │
│     └── 数据权限是否正确？                                               │
│                                                                          │
│  2. 检查作业配置                                                         │
│     ├── Schema 是否正确？                                                │
│     ├── 表配置是否正确？                                                 │
│     └── 资源是否足够？                                                   │
│                                                                          │
│  3. 检查输出                                                             │
│     ├── Segment 是否生成？                                               │
│     ├── 上传是否成功？                                                   │
│     └── 元数据是否更新？                                                 │
│                                                                          │
│  解决方案：                                                              │
│  ─────────────────                                                      │
│  ├── 检查数据格式和编码                                                  │
│  ├── 验证 Schema 和表配置                                                │
│  ├── 增加作业资源                                                        │
│  └── 手动重试                                                            │
│                                                                          │
│  诊断命令：                                                              │
│  ─────────────────                                                      │
│  -- 查看作业日志                                                         │
│  yarn logs -applicationId <application_id>                               │
│                                                                          │
│  -- 手动测试 Segment 生成                                                │
│  bin/pinot-admin.sh CreateSegment \
│    -schemaFile schema.json \
│    -tableConfigFile table.json \
│    -dataDir /path/to/data \
│    -outDir /path/to/output                                               │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3. 集群问题

### 3.1 Server 宕机

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Server 宕机排查                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  症状：Server 节点不可用                                                 │
│                                                                          │
│  排查步骤：                                                              │
│  ─────────────────                                                      │
│  1. 检查进程状态                                                         │
│     ├── 进程是否运行？                                                   │
│     ├── 端口是否监听？                                                   │
│     └── 日志是否有错误？                                                 │
│                                                                          │
│  2. 检查资源                                                             │
│     ├── 内存是否耗尽？                                                   │
│     ├── 磁盘是否满？                                                     │
│     └── CPU 是否饱和？                                                   │
│                                                                          │
│  3. 检查网络                                                             │
│     ├── 网络是否连通？                                                   │
│     ├── DNS 是否正常？                                                   │
│     └── 防火墙是否阻止？                                                 │
│                                                                          │
│  解决方案：                                                              │
│  ─────────────────                                                      │
│  ├── 重启 Server                                                         │
│  ├── 释放资源（内存、磁盘）                                              │
│  ├── 检查网络配置                                                        │
│  └── 检查 JVM 配置                                                       │
│                                                                          │
│  诊断命令：                                                              │
│  ─────────────────                                                      │
│  -- 检查进程                                                             │
│  ps -ef | grep pinot                                                     │
│                                                                          │
│  -- 检查端口                                                             │
│  netstat -tlnp | grep 8098                                               │
│                                                                          │
│  -- 查看日志                                                             │
│  tail -f /var/log/pinot/pinot-server.log                                 │
│                                                                          │
│  -- 检查内存                                                             │
│  free -h                                                                 │
│                                                                          │
│  -- 检查磁盘                                                             │
│  df -h                                                                   │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        查询慢排查                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  症状：查询响应时间长                                                    │
│                                                                          │
│  排查步骤：                                                              │
│  ─────────────────                                                      │
│  1. 分析查询计划                                                         │
│     ├── 使用了哪些索引？                                                 │
│     ├── 扫描了多少数据？                                                 │
│     └── 哪些 Segment 参与了查询？                                        │
│                                                                          │
│  2. 检查索引                                                             │
│     ├── 索引是否已构建？                                                 │
│     ├── 索引类型是否合适？                                               │
│     └── 索引是否被使用？                                                 │
│                                                                          │
│  3. 检查资源                                                             │
│     ├── Server CPU 使用率                                                │
│     ├── 内存命中率                                                       │
│     └── 磁盘 IO                                                          │
│                                                                          │
│  解决方案：                                                              │
│  ─────────────────                                                      │
│  ├── 添加合适的索引                                                      │
│  ├── 优化查询条件                                                        │
│  ├── 使用 Star-Tree 索引                                                 │
│  ├── 增加 Server 资源                                                    │
│  └── 预热 Segment                                                        │
│                                                                          │
│  诊断命令：                                                              │
│  ─────────────────                                                      │
│  -- 查看查询统计                                                         │
│  curl -X POST http://pinot-broker:8099/query/sql \
```

### 4.2 内存不足

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        内存不足排查                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  症状：OOM 错误，进程被杀死                                              │
│                                                                          │
│  排查步骤：                                                              │
│  ─────────────────                                                      │
│  1. 检查 JVM 内存                                                        │
│     ├── 堆内存使用                                                       │
│     ├── GC 频率                                                          │
│     └── 内存泄漏                                                         │
│                                                                          │
│  2. 检查 Segment 加载                                                    │
│     ├── 加载了多少 Segment？                                             │
│     ├── Segment 大小？                                                   │
│     └── 是否使用了 MMAP？                                                │
│                                                                          │
│  3. 检查查询                                                             │
│     ├── 是否有大查询？                                                   │
│     ├── 并发查询数？                                                     │
│     └── 结果集大小？                                                     │
│                                                                          │
│  解决方案：                                                              │
│  ─────────────────                                                      │
│  ├── 增加 JVM 堆内存                                                     │
│  ├── 使用 G1GC 或 ZGC                                                    │
│  ├── 限制并发查询数                                                      │
│  ├── 卸载不常用 Segment                                                  │
│  └── 优化查询减少内存使用                                                │
│                                                                          │
│  诊断命令：                                                              │
│  ─────────────────                                                      │
│  -- 查看 JVM 内存                                                        │
│  jmap -heap <pid>                                                        │
│                                                                          │
│  -- 查看 GC 日志                                                         │
│  tail -f /var/log/pinot/gc.log                                           │
│                                                                          │
│  -- 生成堆转储                                                           │
│  jmap -dump:format=b,file=/tmp/heap.hprof <pid>                          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 5. 常用诊断命令

### 5.1 集群状态检查

```bash
# 检查集群健康
curl http://pinot-controller:9000/health

# 查看集群信息
curl http://pinot-controller:9000/cluster/info

# 查看表列表
curl http://pinot-controller:9000/tables

# 查看表详情
curl http://pinot-controller:9000/tables/{tableName}

# 查看 Segment 列表
curl http://pinot-controller:9000/tables/{tableName}/segments

# 查看 Broker 路由
curl http://pinot-broker:8099/debug/routingTable
```

### 5.2 Server 状态检查

```bash
# 查看消费状态
curl http://pinot-server:8097/debug/consumers

# 查看查询状态
curl http://pinot-server:8097/debug/queries

# 查看 Segment 状态
curl http://pinot-server:8097/debug/tables/{tableName}

# 查看线程状态
curl http://pinot-server:8097/debug/threads

# 查看 JVM 信息
curl http://pinot-server:8097/debug/jvm
```
