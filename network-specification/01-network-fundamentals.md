# 1. OSI七层模型

## 1.1 模型概述

```
┌─────────────────────────────────────────────────────────────────┐
│                    OSI七层模型                                   │
└─────────────────────────────────────────────────────────────────┘

┌────────┬───────────┬───────────────────┬───────────────────────┐
│ 层次   │ 名称       │ 核心功能           │ 典型设备/协议           │
├────────┼───────────┼───────────────────┼───────────────────────┤
│  第7层  │ 应用层     │ 提供网络服务接口    │ HTTP, FTP, SMTP, DNS   │
├────────┼───────────┼───────────────────┼───────────────────────┤
│  第6层  │ 表示层     │ 数据格式转换/加密  │ TLS, SSL, JPEG, ASCII  │
├────────┼───────────┼───────────────────┼───────────────────────┤
│  第5层  │ 会话层     │ 会话管理/连接维护  │ NetBIOS, RPC, SQL     │
├────────┼───────────┼───────────────────┼───────────────────────┤
│  第4层  │ 传输层     │ 端到端可靠传输     │ TCP, UDP              │
├────────┼───────────┼───────────────────┼───────────────────────┤
│  第3层  │ 网络层     │ IP寻址和路由      │ IP, ICMP, Router       │
├────────┼───────────┼───────────────────┼───────────────────────┤
│  第2层  │ 数据链路层 │ MAC寻址/帧传输    │ Ethernet, Switch      │
├────────┼───────────┼───────────────────┼───────────────────────┤
│  第1层  │ 物理层     │ 比特流传输        │ Cable, Hub, Repeater  │
└────────┴───────────┴───────────────────┴───────────────────────┘

# 记忆口诀: "应表会传网数物" (All People Seem To Need Data Processing)
# 从下往上: Physical → Data Link → Network → Transport → Session → Presentation → Application
```

## 1.2 各层职责详解

```bash
# 第1层 - 物理层: 负责比特的传输
# 设备: 网线、光纤、集线器、Repeater
# 信号: 电信号、光信号、无线电波

# 第2层 - 数据链路层: 负责帧的传输和MAC寻址
# 设备: 交换机、网桥
# 地址: MAC地址 (48位, 如: 00:1A:2B:3C:4D:5E)
# 协议: Ethernet, PPP, HDLC

# 第3层 - 网络层: 负责IP寻址和路由
# 设备: 路由器、三层交换机
# 地址: IP地址 (IPv4: 32位, IPv6: 128位)
# 协议: IP, ICMP, ARP, RIP, OSPF, BGP

# 第4层 - 传输层: 负责端到端的可靠传输
# 协议: TCP (可靠), UDP (不可靠)
# 端口: 16位 (0-65535)
# 知名端口: HTTP(80), HTTPS(443), SSH(22), DNS(53)

# 第5-7层 - 应用层: 负责具体应用服务
# 协议: HTTP, FTP, SMTP, POP3, IMAP, DNS, SSH, Telnet
```

# 2. TCP/IP四层模型

## 2.1 模型对应关系

```
┌─────────────────────────────────────────────────────────────────┐
│                    TCP/IP四层模型 vs OSI七层                      │
└─────────────────────────────────────────────────────────────────┘

┌────────────────┐     ┌────────┬───────────┐
│   应用层        │◄───►│  第7层  │ 应用层     │
│   (Application) │     │  第6层  │ 表示层     │
│                │     │  第5层  │ 会话层     │
├────────────────┤     ├────────┼───────────┤
│   传输层        │◄───►│  第4层  │ 传输层     │
│   (Transport)  │     ├────────┼───────────┤
│                │     │  第3层  │ 网络层     │
├────────────────┤     ├────────┼───────────┤
│   网络层        │◄───►│  第2层  │ 数据链路层 │
│   (Internet)   │     ├────────┼───────────┤
│                │     │  第1层  │ 物理层     │
├────────────────┤     └────────┴───────────┘
│   网络接口层    │
│   (Link)       │
└────────────────┘

# TCP/IP四层:
# 1. 应用层: HTTP, FTP, DNS, SMTP...
# 2. 传输层: TCP, UDP
# 3. 网络层: IP, ICMP, ARP
# 4. 网络接口层: Ethernet, PPP
```

## 2.2 数据单元

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据单元对比                                   │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ 应用层         │ Data (数据)                                     │
├─────────────────────────────────────────────────────────────────┤
│ 传输层         │ Segment (TCP) / Datagram (UDP)                 │
│                │ [Src Port][Dst Port][Sequence][Ack][Flags]    │
├─────────────────────────────────────────────────────────────────┤
│ 网络层         │ Packet (数据包)                                │
│                │ [Src IP][Dst IP][Protocol][TTL]                │
├─────────────────────────────────────────────────────────────────┤
│ 数据链路层     │ Frame (帧)                                     │
│                │ [Dst MAC][Src MAC][Type][Data][CRC]           │
├─────────────────────────────────────────────────────────────────┤
│ 物理层         │ Bits (比特流)                                  │
└─────────────────────────────────────────────────────────────────┘

# 封装过程:
# 应用层数据 → TCP/UDP头 → IP头 → Ethernet头 → 比特
#   Data        Segment      Packet     Frame      Bits

# 解封装过程:
# 比特 → Ethernet帧 → IP包 → TCP段 → 应用数据
```

# 3. 数据封装与解封装

## 3.1 封装过程

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据封装过程                                   │
└─────────────────────────────────────────────────────────────────┘

发送端 (封装):

┌─────────────────────────────────────────────────────────────────┐
│  应用层: "GET /index.html HTTP/1.1\r\nHost: example.com\r\n"  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ 添加应用层头
┌─────────────────────────────────────────────────────────────────┐
│  传输层: TCP Segment                                          │
│  [Src Port: 54321][Dst Port: 80][Seq: 100][Ack: 0][Flags]     │
│  [Data: GET /index.html...]                                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ 添加网络层头
┌─────────────────────────────────────────────────────────────────┐
│  网络层: IP Packet                                             │
│  [Src IP: 192.168.1.100][Dst IP: 93.184.216.34][Protocol: 6]   │
│  [TCP Segment]                                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ 添加链路层头
┌─────────────────────────────────────────────────────────────────┐
│  数据链路层: Ethernet Frame                                     │
│  [Dst MAC: 00:11:22:33:44:55][Src MAC: 66:77:88:99:AA:BB]      │
│  [Type: 0x0800][IP Packet][CRC: 0x12345678]                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ 转换为比特
    10101010101010101010101010101010101010101010101010...
```

## 3.2 解封装过程

```
接收端 (解封装):

    10101010101010101010101010101010101010101010101010...
                              │
                              ▼ 校验CRC, 移除Ethernet头
┌─────────────────────────────────────────────────────────────────┐
│  数据链路层: 检查MAC地址, 移除帧头                               │
│  [Type: 0x0800] → 交给网络层处理                                │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ 移除IP头
┌─────────────────────────────────────────────────────────────────┐
│  网络层: 检查IP头部, 确认目标IP是自己                           │
│  [Protocol: 6] → 交给传输层处理                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ 移除TCP头
┌─────────────────────────────────────────────────────────────────┐
│  传输层: 检查端口, 按序列号重组数据                              │
│  [Data: GET /index.html...] → 交给应用层                        │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  应用层: HTTP协议解析                                           │
│  GET /index.html HTTP/1.1                                      │
└─────────────────────────────────────────────────────────────────┘
```

# 4. 网络设备与层次对应

## 4.1 设备层次图

```
┌─────────────────────────────────────────────────────────────────┐
│                    网络设备与OSI层次对应                         │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  第7层  │  应用层     │ Web服务器, 邮件服务器, DNS服务器        │
├─────────────────────────────────────────────────────────────────┤
│  第6层  │  表示层     │ TLS/SSL加密代理                          │
├─────────────────────────────────────────────────────────────────┤
│  第5层  │  会话层     │ 会话管理设备                             │
├─────────────────────────────────────────────────────────────────┤
│  第4层  │  传输层     │ 四层交换机 (Load Balancer)              │
├─────────────────────────────────────────────────────────────────┤
│  第3层  │  网络层     │ 路由器, 三层交换机                      │
├─────────────────────────────────────────────────────────────────┤
│  第2层  │  数据链路层 │ 交换机, 网桥                            │
├─────────────────────────────────────────────────────────────────┤
│  第1层  │  物理层     │ 集线器, 光纤, 双绞线, Repeater          │
└─────────────────────────────────────────────────────────────────┘

# 理解:
# - 集线器是第1层设备, 简单放大/转发信号
# - 交换机是第2层设备, 根据MAC地址转发帧
# - 路由器是第3层设备, 根据IP地址转发数据包
# - 四层交换机是第4层设备, 根据端口号做负载均衡
```

## 4.2 MAC地址表 vs 路由表

```bash
# 交换机MAC地址表 (CAM表)
# 动态学习, 根据源MAC记录端口

show mac address-table
# VLAN  Mac Address      Type      Ports
# ----  -----------      --------  -----
#   1    00:11:22:33:44:55    DYNAMIC  Gi0/1
#   1    66:77:88:99:AA:BB    DYNAMIC  Gi0/2

# 路由器路由表
# 静态/动态学习, 根据目的网络转发

ip route show
# default via 192.168.1.1 dev eth0  proto static
# 10.0.0.0/24 via 192.168.1.254 dev eth0  proto static

# 交换机转发逻辑:
# 1. 收到帧, 学习源MAC
# 2. 查找目标MAC:
#    - 在表中 → 从对应端口转发
#    - 不在表中 → 泛洪 (广播)
# 3. 目标MAC是广播地址 → 泛洪

# 路由器转发逻辑:
# 1. 收到IP包, 检查TTL
# 2. 查找路由表:
#    - 找到路由 → 从对应接口转发
#    - 没找到 → 发送给默认网关
# 3. TTL减1, 重新封装, 发送
```

# 5. 网络通信实战

## 5.1 查看网络配置

```bash
# 查看所有网络接口
ip link show
# 1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN
#    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
# 2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast
#    link/ether 00:11:22:33:44:55 brd ff:ff:ff:ff:ff:ff

# 查看IP地址
ip addr show
# inet 192.168.1.100/24 scope global eth0

# 查看路由表
ip route show

# 查看ARP表
ip neigh show
# 192.168.1.1 dev eth0 lladdr 00:11:22:33:44:55 REACHABLE

# 查看网络连接统计
ss -s
# Total: 186 (kernel: 194)
# TCP:   12 (estab: 5, closed: 0, orphaned: 0, synrecv: 0)
```

## 5.2 抓包分析

```bash
# 抓取所有包
sudo tcpdump -i eth0

# 抓取特定主机
sudo tcpdump -i eth0 host 8.8.8.8

# 抓取特定端口
sudo tcpdump -i eth0 port 80

# 抓取HTTP包
sudo tcpdump -i eth0 port 80 -A

# 抓取并保存到文件 (用于Wireshark分析)
sudo tcpdump -i eth0 -w capture.pcap

# 读取抓包文件
tcpdump -r capture.pcap

# 抓取特定协议的包
sudo tcpdump -i eth0 icmp
sudo tcpdump -i eth0 tcp
sudo tcpdump -i eth0 udp

# 抓取特定网段的包
sudo tcpdump -i eth0 net 192.168.1.0/24
```

## 5.3 网络路径分析

```bash
# 路由追踪 (使用ICMP)
traceroute 8.8.8.8

# 路由追踪 (使用TCP, 穿透防火墙)
traceroute -T -p 80 8.8.8.8

#mtr - 实时路由追踪
mtr 8.8.8.8

# 查看DNS解析
dig example.com
nslookup example.com
host example.com

# 查看WHOIS信息
whois example.com
```

# 6. 网络协议基础

## 6.1 Ethernet帧格式

```
┌─────────────────────────────────────────────────────────────────┐
│                    Ethernet II 帧格式                           │
└─────────────────────────────────────────────────────────────────┘

┌──────────┬──────────┬──────────┬──────────────────┬──────────┐
│  前导码   │ 目标MAC  │ 源MAC    │   类型/长度        │  数据    │ CRC
│  8字节    │ 6字节    │ 6字节    │   2字节           │ 46-1500B │ 4字节
└──────────┴──────────┴──────────┴──────────────────┴──────────┘

# 前导码: 7字节 10101010 + 1字节 10101011 (帧起始定界符)
# 目标MAC: 接收方MAC地址
# 源MAC: 发送方MAC地址
# 类型: 0x0800=IPv4, 0x0806=ARP, 0x86DD=IPv6
# 数据: 最少46字节, 最大1500字节 (MTU)
# CRC: 循环冗余校验

# 帧最小64字节, 最大1518字节 (不含前导码)
```

## 6.2 IP头格式

```
┌─────────────────────────────────────────────────────────────────┐
│                    IPv4 头格式                                   │
└─────────────────────────────────────────────────────────────────┘

┌────┬─────┬────────┬────────────────┬───────────────────────────┐
│Ver │ IHL │  TOS   │    Total Len   │      Identification       │
│ 4b │ 4b  │  1B    │      2B        │           2B             │
├────┼─────┼────────┼────────────────┼───────────────────────────┤
│Flags│ Frag Offset  │     TTL       │      Protocol            │
│ 3b │    13b        │     1B        │         1B               │
├────┼───────────────┼───────────────┼───────────────────────────┤
│   Header Checksum  │       Source IP Address                 │
│        2B          │              4B                           │
├─────────────────────┼───────────────────────────────────────────┤
│   Destination IP Address                                      │
│                         4B                                      │
├─────────────────────────────────────────────────────────────────┤
│                    Options (可选)                               │
└─────────────────────────────────────────────────────────────────┘

# Ver: 版本 (4=IPv4, 6=IPv6)
# IHL: 头长度 (通常5, 表示20字节)
# TOS: 服务类型 (IP优先级/DSCP)
# Total Len: 总长度 (头+数据)
# Identification: 标识 (分片重组用)
# Flags: 标志 (DF=不分片, MF=更多分片)
# TTL: 生存时间 (最大255, 每跳减1)
# Protocol: 协议 (1=ICMP, 6=TCP, 17=UDP)
# Header Checksum: 头校验和
# Source IP: 源IP
# Dest IP: 目标IP
```

## 本章小结

- **OSI七层模型**从下到上: 物理层→数据链路层→网络层→传输层→会话层→表示层→应用层
- **TCP/IP四层模型**: 应用层、传输层、网络层、网络接口层
- **数据封装**: 应用数据→TCP头→IP头→Ethernet头→比特
- **网络设备层次**: 集线器(第1层)→交换机(第2层)→路由器(第3层)
- **MAC地址**用于局域网寻址, **IP地址**用于全网寻址
- **TTL**限制数据包生存周期, 防止环路

**关键命令回顾:**

```bash
# 查看配置
ip link show, ip addr show, ip route show, ss -s

# 抓包分析
tcpdump -i eth0, tcpdump -i eth0 host x.x.x.x, tcpdump -i eth0 port 80

# 路径分析
traceroute, mtr, dig, nslookup

# 查看ARP表
ip neigh show
```
