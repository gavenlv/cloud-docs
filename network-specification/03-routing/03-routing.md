# 1. 路由原理概述

## 1.1 路由器工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    路由器工作原理                                │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                        路由器                                    │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐         │
│  │   接口1     │   │   接口2     │   │   接口3     │         │
│  │  eth0      │   │  eth1      │   │  eth2      │         │
│  │ 192.168.1.1│   │ 10.0.0.1   │   │ 172.16.0.1 │         │
│  └──────┬──────┘   └──────┬──────┘   └──────┬──────┘         │
│         │                  │                  │                 │
│         └──────────────────┴──────────────────┘                 │
│                            │                                   │
│                    ┌───────┴───────┐                          │
│                    │   路由引擎    │                          │
│                    │ (路由表/FIB)  │                          │
│                    └───────────────┘                          │
└─────────────────────────────────────────────────────────────────┘

# 路由器转发流程:
# 1. 收到IP包
# 2. 检查IP头 (TTL减1, 校验和)
# 3. 查找路由表 (FIB - Forwarding Information Base)
# 4. 匹配路由:
#    - 找到 → 从对应接口转发
#    - 没找到 → 发送给默认网关
# 5. 重新封装 (更换MAC地址)
# 6. 发送

# 核心概念:
# - 路由表: 告诉路由器数据包应该发送到哪
# - FIB: 路由表的优化版本, 用于快速查找
# - ARP: 解析下一跳MAC地址
```

## 1.2 路由表结构

```bash
# 查看Linux路由表
ip route show
# 或
route -n

# 示例输出:
# default via 192.168.1.1 dev eth0 proto static
# 10.0.0.0/8 via 10.0.0.1 dev eth1 proto static
# 172.16.0.0/16 via 192.168.1.254 dev eth0 proto static
# 192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.100

# 字段说明:
# default: 默认路由 (0.0.0.0/0)
# via 192.168.1.1: 下一跳地址
# dev eth0: 出接口
# proto static: 路由协议 (static=静态, kernel=内核发现)
# scope link: 作用域 (link=本地链路, global=全局)
# src 192.168.1.100: 源IP地址

# 详细查看路由表
ip route show table all
ip route show table local
ip route show table main
```

# 2. 路由查找过程

## 2.1 最长前缀匹配

```
┌─────────────────────────────────────────────────────────────────┐
│                    最长前缀匹配 (Longest Prefix Match)           │
└─────────────────────────────────────────────────────────────────┘

# 路由表:
# 0.0.0.0/0          → default gateway (匹配最少)
# 192.168.0.0/16     → via 10.0.0.1
# 192.168.1.0/24     → via 192.168.1.1
# 192.168.1.100/32   → local (匹配最精确)

# 目标IP: 192.168.1.100

匹配过程:
┌─────────────────────────────────────────────────────┐
│ 0.0.0.0/0          → 匹配0位    ✗                  │
│ 192.168.0.0/16     → 匹配16位   ✗                  │
│ 192.168.1.0/24     → 匹配24位   ✓ (选中)          │
│ 192.168.1.100/32   → 匹配32位   ✓ (更精确,但不在表中)│
└─────────────────────────────────────────────────────┘

# 结果: 使用 192.168.1.0/24 这条路由

# 目标IP: 10.20.30.40

匹配过程:
┌─────────────────────────────────────────────────────┐
│ 0.0.0.0/0          → 匹配0位    ✓ (只有这条匹配)   │
└─────────────────────────────────────────────────────┘

# 结果: 使用默认路由
```

## 2.2 路由优先级

```bash
# Linux路由优先级 (数字越小优先级越高)
ip rule show
# 0:      from all lookup local
# 32766:  from all lookup main
# 32767:  from all lookup default

# 路由表优先级顺序:
# 1. local (local路由, 最高优先)
# 2. main (普通路由, 默认)
# 3. default (备用路由)

# 路由类型优先级 (使用ip route时):
# 1. Broadcast (广播)
# 2. Local (本地)
# 3. Unicast (单播) - 包括:
#    - unreachable
#    - blackhole
#    - prohibit
#    - throw
#    - nat
#    - via (普通路由)

# 自定义路由表优先级
ip rule add from 192.168.1.0/24 table custom priority 100
```

# 3. 静态路由

## 3.1 静态路由配置

```bash
# 添加静态路由
ip route add 10.0.0.0/8 via 192.168.1.1 dev eth0

# 添加默认路由
ip route add default via 192.168.1.1 dev eth0
# 或
ip route add 0.0.0.0/0 via 192.168.1.1 dev eth0

# 删除路由
ip route del 10.0.0.0/8 via 192.168.1.1 dev eth0

# 添加主机路由
ip route add 192.168.1.100/32 via 192.168.1.1 dev eth0

# 添加黑洞路由 (丢弃流量)
ip route add blackhole 10.0.0.0/8

# 添加不可达路由 (发送ICMP不可达)
ip route add prohibit 172.16.0.0/12

# 查看特定路由
ip route get 8.8.8.8
# 8.8.8.8 via 192.168.1.1 dev eth0 src 192.168.1.100

# 持久化静态路由 (重启后保留)
# Debian/Ubuntu:
echo "10.0.0.0/8 via 192.168.1.1" >> /etc/network/interfaces

# RHEL/CentOS:
echo "10.0.0.0/8 via 192.168.1.1 dev eth0" >> /etc/sysconfig/network-scripts/route-eth0
```

## 3.2 静态路由场景

```
┌─────────────────────────────────────────────────────────────────┐
│                    静态路由应用场景                              │
└─────────────────────────────────────────────────────────────────┘

# 场景1: 双网卡服务器
┌──────────┐
│  服务器   │
│ eth0: 192.168.1.100/24  ←────→ 192.168.1.0/24 网络
│ eth1: 10.0.0.100/8      ←────→ 10.0.0.0/8 网络
└──────────┘

# 需要添加:
ip route add 10.0.0.0/8 dev eth1
ip route add 192.168.1.0/24 dev eth0

# 场景2: 指定网关
┌──────────┐     ┌──────────┐     ┌──────────┐
│  PC     │────►│  Router  │────►│ Internet │
│         │     │  eth0    │     │          │
│ 192.168.│     │ 192.168.1│     │          │
│   1.100 │     │    .1    │     │          │
└──────────┘     └──────────┘     └──────────┘

# PC需要默认路由:
ip route add default via 192.168.1.1
```

# 4. 动态路由协议

## 4.1 路由协议分类

```
┌─────────────────────────────────────────────────────────────────┐
│                    动态路由协议分类                              │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│            动态路由协议                 │
├──────────────────┬──────────────────────┤
│  IGP (内部网关协议)│  EGP (外部网关协议) │
│  - RIP           │  - BGP              │
│  - OSPF          │                      │
│  - IS-IS         │                      │
│  - EIGRP         │                      │
└──────────────────┴──────────────────────┘
       │                    │
       ▼                    ▼
┌─────────────────────────────────────────┐
│           路由算法分类                   │
├──────────────────┬──────────────────────┤

# 主要协议对比:
```

## 4.2 RIP协议

```
┌─────────────────────────────────────────────────────────────────┐
│                    RIP (Routing Information Protocol)           │
└─────────────────────────────────────────────────────────────────┘

# 工作原理:
# 1. 每30秒广播完整路由表
# 2. 收到路由后, 跳数+1
# 3. 如果新路由跳数更少, 更新
# 4. 跳数最大15, 16=不可达

# 特点:
# - 简单, 适合小型网络
# - 收敛慢, 有路由环路风险
# - 跳数限制15, 不适合大型网络

# RIPng (IPv6版)
# 配置:
router rip
 version 2
 network 192.168.1.0
```

## 4.3 OSPF协议

```
┌─────────────────────────────────────────────────────────────────┐
│                    OSPF (Open Shortest Path First)             │
└─────────────────────────────────────────────────────────────────┘

# 工作原理:
# 1. 建立邻居关系 (Hello包)
# 2. 交换链路状态 (LSA)
# 3. 构建链路状态数据库 (LSDB)
# 4. 使用Dijkstra算法计算最短路径
# 5. 生成最优路由

# 区域 (Area) 设计:
┌─────────────────────────────────────────────────────────────────┐
│                      Area 0 (骨干区域)                          │
│    ┌─────────────────────────────────────────────────┐          │
│    │                                                  │          │
│    │   ┌──────┐    ┌──────┐    ┌──────┐           │          │
│    │   │Area 0│◄──►│Area 0│◄──►│Area 0│           │          │
│    │   │      │    │(Backbone)│   │      │           │          │
│    │   └──┬───┘    └─────────┘    └──┬───┘           │          │
│    │      │                          │               │          │
│    │   ┌──┴───┐                   ┌──┴───┐          │          │
│    │   │Area 1│                   │Area 2│          │          │
│    │   │Stub  │                   │ NSSA │          │          │
│    │   └──────┘                   └──────┘          │          │
└────┴──────────────────────────────────────────────────┴──────────┘

# OSPF成本计算:
# 成本 = 参考带宽(100Mbps) / 接口带宽
# 10Mbps链路 → 成本 = 10
# 100Mbps链路 → 成本 = 1
# 1Gbps链路 → 成本 = 0.1 (但最小为1)

# OSPF配置:
router ospf 1
 network 192.168.1.0 0.0.0.255 area 0
 network 10.0.0.0 0.255.255.255 area 0
```

## 4.4 BGP协议

```
┌─────────────────────────────────────────────────────────────────┐
│                    BGP (Border Gateway Protocol)               │
└─────────────────────────────────────────────────────────────────┘

# BGP特点:
# - 路径矢量协议
# - 用于AS(自治系统)之间
# - 互联网主干协议
# - 支持复杂的路由策略

# AS号 (1-65535):
# - 公有AS: 1-64511
# - 私有AS: 64512-65535

# BGP邻居关系:
┌─────────┐                           ┌─────────┐
│   AS1   │◄──── eBGP (不同AS) ────►│   AS2   │
│         │                           │         │
│  ┌────┐ │◄──── iBGP (同一AS) ────►│  ┌────┐ │
│  │ BGP│ │                           │  │ BGP│ │
│  └────┘ │                           │  └────┘ │
└─────────┘                           └─────────┘

# BGP路由属性:
# - AS-Path: 经过的AS序列
# - Next_Hop: 下一跳
# - Local_Pref: 本地优先级
# - MED: 多出口鉴别器
# - Origin: 起源

# BGP配置:
router bgp 65001
 neighbor 192.168.1.1 remote-as 65002
 network 10.0.0.0 mask 255.0.0.0
```

# 5. 路由策略

## 5.1 策略路由

```bash
# 基于源地址的策略路由
ip rule add from 192.168.1.0/24 table 100 priority 100

# 创建自定义路由表
echo "100 custom" >> /etc/iproute2/rt_tables

# 添加路由到自定义表
ip route add default via 10.0.0.1 dev eth1 table custom

# 查看路由表
ip route show table custom

# 基于源地址的完整配置:
# 1. 创建路由表
echo "200 wan1" >> /etc/iproute2/rt_tables
echo "201 wan2" >> /etc/iproute2/rt_tables

# 2. 添加默认路由
ip route add default via 203.0.113.1 dev eth0 table wan1
ip route add default via 198.51.100.1 dev eth1 table wan2

# 3. 添加策略路由
ip rule add from 192.168.1.0/24 table wan1 priority 100
ip rule add from 192.168.2.0/24 table wan2 priority 101

# 4. 查看
ip rule show
ip route show table wan1
ip route show table wan2
```

## 5.2 负载均衡

```bash
# 静态负载均衡 (基于轮询)
ip route add default scope global \
    nexthop via 192.168.1.1 dev eth0 weight 1 \
    nexthop via 10.0.0.1 dev eth1 weight 1

# 基于策略的负载均衡
# 不同源地址走不同链路
ip rule add from 192.168.1.0/24 table wan1
ip rule add from 192.168.2.0/24 table wan2

# 查看ECMP (等价多路径)
ip route show
# default via 192.168.1.1 dev eth0
#         nexthop via 10.0.0.1 dev eth1 weight 1
#         nexthop via 192.168.1.1 dev eth0 weight 1

# 启用ECMP
sysctl -w net.ipv4.conf.all.rp_filter=1
```

# 6. NAT与路由

## 6.1 NAT原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    NAT (Network Address Translation)            │
└─────────────────────────────────────────────────────────────────┘

# NAT类型:
# 1. 静态NAT (1:1映射)
#    内部IP ↔ 外部IP (固定映射)
#
# 2. 动态NAT (Pool-based)
#    内部IP ↔ 外部IP (从地址池动态分配)
#
# 3. PAT/NAT Overload (端口地址转换)
#    多个内部IP ↔ 一个外部IP (通过端口区分)

# PAT工作原理:
内部: 192.168.1.100:54321 → 外部: 203.0.113.1:54321
      192.168.1.101:54321 → 外部: 203.0.113.1:54322
      (源端口变化, 区分不同内部主机)

# NAT表:
# 新建连接时, 路由器记录:
# 内部IP:端口 ↔ 外部IP:端口
```

## 6.2 iptables NAT配置

```bash
# SNAT (源地址转换) - 发出流量
iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -o eth0 -j MASQUERADE

# DNAT (目标地址转换) - 流入流量
iptables -t nat -A PREROUTING -d 203.0.113.1 -p tcp --dport 80 \
         -j DNAT --to-destination 192.168.1.100:80

# 端口转发
iptables -t nat -A PREROUTING -p tcp --dport 8080 \
         -j REDIRECT --to-port 80

# 查看NAT表
iptables -t nat -L -n -v

# 查看NAT连接跟踪
conntrack -L
```

## 本章小结

- **路由表**是路由器转发数据包的核心, 通过最长前缀匹配选择路由
- **静态路由**需要手动配置, 适合简单网络
- **动态路由协议**自动学习和分发路由信息
- **RIP**是距离矢量协议, 跳数有限制
- **OSPF**是链路状态协议, 支持区域划分, 适合中大型网络
- **BGP**是路径矢量协议, 用于AS之间, 是互联网主干协议
- **策略路由**可以基于源地址、服务类型等做复杂路由决策
- **NAT**通过修改IP头实现地址转换, 解决公网IP不足问题

**关键命令回顾:**

```bash
# 查看路由
ip route show, route -n, ip rule show

# 静态路由
ip route add/del, ip route get

# 动态路由
# OSPF
router ospf 1, show ip ospf neighbor, show ip ospf route

# BGP
router bgp, show ip bgp, show ip bgp neighbor

# NAT
iptables -t nat -L, conntrack -L

# 诊断
traceroute, mtr, ip route get
```
