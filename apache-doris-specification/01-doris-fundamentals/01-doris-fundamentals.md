# Doris基础

## 概述

Apache Doris是一个现代化的MPP分析型数据库产品，仅需亚秒级响应时间即可获得查询结果。它支持高并发点查询和高吞吐复杂分析场景，支持向量化的执行器，支持MySQL协议，以及标准SQL。

## 核心特性

### 1. 高性能

- 向量化执行器：利用CPU的SIMD指令加速计算
- 列式存储：压缩比高，扫描效率高
- 物化视图：自动加速查询

### 2. 高可用

- 多副本机制：数据多副本存储
- 自动恢复：节点故障自动恢复
- 负载均衡：FE节点可水平扩展

### 3. 易用性

- MySQL协议：兼容MySQL客户端
- 标准SQL：支持完整的SQL语法
- 实时更新：支持实时数据更新

## 架构组件

### Frontend (FE)

- 负责SQL解析和执行计划生成
- 负责查询调度和协调
- 支持高可用部署（Leader/Follower/Observer）

### Backend (BE)

- 负责数据存储和查询执行
- 列式存储引擎
- 向量化执行引擎

### Broker

- 负责访问外部数据源（如HDFS、S3）
- 可选组件

## 适用场景

1. **实时数据分析**：报表分析、用户行为分析
2. **即席查询**：数据探索、快速分析
3. **多维分析**：OLAP分析、BI报表
4. **日志分析**：日志检索、日志统计

## 基本概念

### 集群架构

```
┌─────────────────────────────────────────────────┐
│                   FE (Leader)                   │
│         ┌──────────┬──────────┐                │
│         │  FE-1   │   FE-2  │   FE-3         │
│         │(Leader) │(Follower)│ (Observer)    │
└─────────┴──────────┴──────────┴────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│              BE (Backend Nodes)                 │
│         ┌──────────┬──────────┐                │
│         │  BE-1   │   BE-2  │   BE-3         │
│         │         │         │                │
└─────────┴──────────┴──────────┴────────────────┘
```

### Tablet和Partition

- **Tablet**：数据分片，最小数据管理单元
- **Partition**：分区，用于数据管理

### 数据模型

- **Duplicate模型**：保留完整的原始数据
- **Aggregate模型**：自动聚合相同Key的数据
- **Unique模型**：保证Key唯一性

## 相关命令

查看Doris集群状态：

```bash
SHOW FRONTENDS;
SHOW BACKENDS;
SHOW PROC '/frontends';
SHOW PROC '/backends';
```
