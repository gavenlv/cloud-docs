# GCP存储服务深入解析

## 本章概述

GCP提供完整的存储服务解决方案，涵盖对象存储、块存储和文件存储三大类型。本章深入讲解每种存储服务的原理、适用场景、架构设计和实战操作，帮助你全面掌握GCP存储技术。

## 学习目标

- 深入理解GCP存储服务架构设计理念
- 掌握Cloud Storage对象存储的原理和操作
- 掌握Persistent Disk块存储的使用和管理
- 掌握Filestore文件存储的配置和挂载
- 理解不同存储类型的选型原则
- 掌握Windows环境下的存储操作命令

### 2.3 存储桶操作

```powershell
# ============================================================
# 创建存储桶
# ============================================================

# 基础创建
gsutil mb gs://my-bucket-name

# 参数说明:
# ├── mb: make bucket 创建存储桶
# ├── -l: 指定区域 (US, EU, ASIA)
# ├── -c: 默认存储类别
# └── -b: 指定位置策略

# 完整示例
gsutil mb -l us-central1 -c STANDARD gs://my-project-bucket

# Windows PowerShell变量使用
$PROJECT_ID = "my-project-id"
$REGION = "us-central1"
$STORAGE_CLASS = "STANDARD"

gsutil mb -l $REGION -c $STORAGE_CLASS gs://"$PROJECT_ID-bucket"

# 为什么指定区域？
# ├── 数据主权合规
# ├── 降低延迟（靠近用户）
# └── 优化成本


# ============================================================
# 存储桶配置
# ============================================================

# 设置默认存储类别
gsutil defstorageclass set NEARLINE gs://my-bucket

# 设置版本控制（版本控制是什么？为什么需要？）
# ├── 作用: 保留对象的历史版本
# ├── 为什么: 防止误删除、满足合规要求
# └── 场景: 重要文件、协作编辑
gsutil versioning set on gs://my-bucket

# 设置生命周期规则（自动化管理）
# 为什么需要生命周期规则？
# └── 自动归档/删除，降低成本
gsutil lifecycle set lifecycle-config.json gs://my-bucket

# lifecycle-config.json 示例:
# {
#   "rule": [
#     {
#       "action": {"type": "SetStorageClass", "storageClass": "NEARLINE"},
#       "condition": {"age": 90}
#     },
#     {
#       "action": {"type": "Delete"},
#       "condition": {"age": 365}
#     }
#   ]
# }

# 设置访问控制（ACL）
# 为什么需要ACL？
# └── 控制谁可以读写桶内对象
gsutil acl ch -u allUsers:R gs://my-bucket/public-read
gsutil acl ch -u user@example.com:WRITE gs://my-bucket


# ============================================================
# 列出存储桶
# ============================================================
gsutil ls

# 列出所有桶
gsutil ls -L gs://  # -L 显示详细信息的

# 列出桶内对象
gsutil ls gs://my-bucket/

# 列出特定前缀的对象
gsutil ls gs://my-bucket/prefix/


# ============================================================
# 删除存储桶
# ============================================================
# 必须先清空桶内所有对象
gsutil rm gs://my-bucket/**

# 然后删除桶
gsutil rb gs://my-bucket
```

### 2.4 对象操作

```powershell
# ============================================================
# 上传对象
# ============================================================

# 上传单个文件
# 为什么用cp而不是mv？
# └── cp保留原文件，mv会删除原文件
gsutil cp myfile.txt gs://my-bucket/

# 上传整个目录
gsutil cp -r ./mydir gs://my-bucket/

# 上传并设置内容类型
gsutil cp index.html gs://my-bucket/ -h "Content-Type:text/html"

# 上传时设置缓存控制
# 为什么需要缓存控制？
# └── 优化CDN性能，减少重复下载
gsutil cp image.png gs://my-bucket/ -h "Cache-Control:public, max-age=31536000"

# 上传并加密（满足合规要求）
# 为什么需要加密？
# └── 保护敏感数据，即使泄露也无法读取
gsutil cp sensitive.csv gs://my-bucket/ -h "x-goog-meta-encryption:aes256"


# ============================================================
# 下载对象
# ============================================================

# 下载单个文件
gsutil cp gs://my-bucket/myfile.txt ./

# 下载整个目录
gsutil cp -r gs://my-bucket/mydir ./

# 下载并保持对象名称
gsutil cp gs://my-bucket/*.json ./

# 并行下载（加速大文件）
gsutil -m cp -r gs://my-bucket/large-dir ./

# -m: 多线程并行操作
# 适用场景: 大批量上传/下载


# ============================================================
# 复制对象
# ============================================================

# 桶内复制
gsutil cp gs://my-bucket/source.txt gs://my-bucket/destination.txt

# 跨桶复制
gsutil cp gs://bucket1/file.txt gs://bucket2/folder/

# 复制并更改存储类别
gsutil cp gs://my-bucket/old.txt gs://my-bucket/new.txt -s NEARLINE

# 复制并设置元数据
gsutil cp local-file.txt gs://my-bucket/ -h "x-goog-meta-owner:admin"


# ============================================================
# 删除对象
# ============================================================

# 删除单个对象
gsutil rm gs://my-bucket/myfile.txt

# 删除特定前缀的所有对象
gsutil rm gs://my-bucket/folder/**

# 删除所有对象（保留桶）
gsutil rm gs://my-bucket/**

# 删除所有对象和版本历史
gsutil rm -a gs://my-bucket/**

# -a: 删除所有版本（需启用版本控制）


# ============================================================
# 元数据操作
# ============================================================

# 查看对象元数据
gsutil ls -L gs://my-bucket/myfile.txt

# 更新元数据
gsutil metadata update -h "Content-Language:zh-CN" gs://my-bucket/myfile.txt

# 添加自定义元数据
gsutil metadata update -h "x-goog-meta-project:myproject" gs://my-bucket/myfile.txt


# ============================================================
# 签名URL（限时访问）
# ============================================================

# 为什么需要签名URL？
# └── 临时授权访问，无需公开对象

# 生成签名URL（有效期1小时）
gsutil signurl -d 1h service-account.json gs://my-bucket/private.txt

# 生成签名URL（指定HTTP方法）
gsutil signurl -m GET -d 1h service-account.json gs://my-bucket/myfile.txt

# Windows PowerShell中使用
$SA_KEY = ".\service-account.json"
$URL = "gs://my-bucket/private.txt"
gsutil signurl -d 1h $SA_KEY $URL

# 参数说明:
# ├── -d: 有效期 (1h=1小时, 7d=7天)
# ├── -m: HTTP方法
# ├── service-account.json: 服务账号密钥
# └── 签名URL包含过期时间，过期后无法访问
```

### 2.5 版本控制与对象保护

```powershell
# ============================================================
# 版本控制
# ============================================================

# 启用版本控制（为什么需要版本控制？）
# ├── 防止误删除：可恢复历史版本
# ├── 协作编辑：保留修改历史
# └── 合规要求：某些行业需要保留历史
gsutil versioning set on gs://my-bucket

# 查看所有版本
gsutil ls -a gs://my-bucket/myfile.txt

# 下载特定版本
gsutil cp gs://my-bucket/myfile.txt#version-id ./

# 删除当前版本（变成历史版本）
gsutil rm gs://my-bucket/myfile.txt

# 删除特定版本（永久删除）
gsutil rm gs://my-bucket/myfile.txt#version-id


# ============================================================
# 锁定对象（不可变存储）
# ============================================================

# 为什么需要对象锁定？
# └── 合规要求：数据一旦写入不可修改/删除
#    金融/医疗/政府数据保留

# 设置保留策略（基于桶）
gsutil retention set 1y gs://my-bucket  # 保留1年

# 锁定对象（锁定后无法删除/修改）
gsutil lockedon set gs://my-bucket/myfile.txt

# 查看保留状态
gsutil ls -L gs://my-bucket/myfile.txt


# ============================================================
# 复制策略（跨区域复制）
# ============================================================

# 为什么需要跨区域复制？
# └── 灾难恢复、降低延迟、合规要求

# 创建复制的目标桶
gsutil mb -l asia-northeast1 gs://my-bucket-replica

# 设置CRR（跨区域复制）
# 需要先启用版本控制
gsutil crr set gs://my-bucket gs://my-bucket-replica

# 配置示例:
# 需要满足的条件:
# ├── 两个桶都启用版本控制
# ├── 源桶启用CRR
# └── 服务账号有目标桶的写入权限
```

### 2.6 CDN与性能优化

```
为什么需要Cloud CDN？

┌─────────────────────────────────────────────────────────────────────────┐
│                    有CDN vs 无CDN                                       │
│                                                                         │
│  无CDN:                                                               │
│  ├── 用户访问 → 源站(可能很远)                                         │
│  ├── 每次访问都需要回源                                                │
│  └── 延迟高、源站压力大                                                │
│                                                                         │
│  有CDN:                                                               │
│  ├── 用户访问 → 最近边缘节点                                           │
│  ├── 边缘节点缓存内容                                                  │
│  ├── 后续访问直接返回缓存                                              │
│  └── 延迟低、源站压力小                                                │
└─────────────────────────────────────────────────────────────────────────┘

缓存策略设计:

┌─────────────────────────────────────────────────────────────────────────┐
│                    缓存过期策略                                         │
├──────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  静态资源 (图片/CSS/JS):                                              │
│  └── 一年: max-age=31536000                                         │
│                                                                       │
│  API响应:                                                             │
│  └── 不缓存或短期: max-age=0                                         │
│                                                                       │
│  用户生成内容:                                                         │
│  └── 验证式缓存: Cache-Control: no-cache                            │
│                                                                       │
└──────────────────────────────────────────────────────────────────────┘
```

```powershell
# ============================================================
# 启用Cloud CDN
# ============================================================

# 通过负载均衡器启用CDN
# 1. 创建后端存储桶
gcloud compute backend-bucket create my-cdn-backend `
    --gcs-bucket-name=my-bucket `
    --enable-cdn `
    --project=$PROJECT_ID

# 2. 创建HTTP(S)负载均衡器
gcloud compute forwarding-rules create my-cdn-lb `
    --global `
    --target-http-proxy=my-proxy `
    --ports=80 `
    --project=$PROJECT_ID


# ============================================================
# 缓存失效
# ============================================================

# 为什么需要缓存失效？
# └── 更新文件后，需要清除CDN缓存

# 失效所有缓存
gcloud compute url-maps invalidate-cdn-cache my-url-map --path="/*"

# 失效特定路径
gcloud compute url-maps invalidate-cdn-cache my-url-map --path="/images/*"

# 注意: 失效可能需要几分钟生效
```

### 4.4 挂载Filestore

```powershell

# ============================================================

# 1. 获取Filestore IP地址
$FILESTORE_IP = gcloud filestore instances describe my-filestore `

df -h | grep filestore

# 添加到 /etc/fstab
$FILESTORE_IP = "10.0.0.5"
"$FILESTORE_IP`:/myvolume /mnt/filestore nfs defaults,_netdev 0 0" | `

sudo mount -a

# 在Windows VM上挂载 (SMB/NFS)

# 2. 启用"NFS服务"和"NFS客户端"
# 3. 使用注册表启用NFS客户端
reg add "HKLM\SOFTWARE\Microsoft\Client\ForNFS\CurrentVersion\Default" /v AnonymousUID /t REG_DWORD /d 0

# 4. 挂载NFS共享

# 权限配置
# ============================================================

# Linux权限设置
sudo chown -R $USER:$USER /mnt/filestore

# 创建共享目录供多用户使用
sudo mkdir -p /mnt/filestore/project1
sudo mkdir -p /mnt/filestore/project2
sudo chown -R project1user:project1group /mnt/filestore/project1
```

```powershell
# ============================================================
# Cloud Storage + Compute Engine
# ============================================================

# 从Cloud Storage下载到VM
gsutil cp gs://my-bucket/data.csv /home/user/data/

# 上传VM数据到Cloud Storage
gsutil cp /home/user/backup.tar.gz gs://my-bucket/backups/

# VM启动时自动挂载Cloud Storage (使用gcsfuse)
# 在/etc/fstab添加:
# gcsfuse my-bucket /mnt/gcs gcsfuse defaults,_netdev,allow_other


# ============================================================
# Cloud Storage + BigQuery
# ============================================================

# 从GCS加载数据到BigQuery
bq load \
    --source_format=CSV \
    myproject:mydataset.mytable \
    gs://my-bucket/data.csv \
    schema.json

# 导出BigQuery结果到GCS
bq extract \
    --destination_format=CSV \
    myproject:mydataset.mytable \
    gs://my-bucket/export/output_*.csv


# ============================================================
# Persistent Disk + GKE
# ============================================================

# 创建StorageClass
kubectl apply -f - <<EOF
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: pd-ssd
provisioner: pd.csi.storage.gke.io
volumeBindingMode: WaitForFirstConsumer
parameters:
  type: pd-ssd
EOF

# 创建PVC
kubectl apply -f - <<EOF
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: pd-ssd
  resources:
    requests:
      storage: 10Gi
EOF

# 在Pod中使用
kubectl apply -f - <<EOF
apiVersion: v1
kind: Pod
metadata:
  name: my-pod
spec:
  containers:
  - name: app
    image: nginx
    volumeMounts:
    - name: data
      mountPath: /data
  volumes:
  - name: data
    persistentVolumeClaim:
      claimName: my-pvc
EOF


# ============================================================
# Filestore + GKE
# ============================================================

# 创建Filestore CSI驱动PV
kubectl apply -f - <<EOF
apiVersion: v1
kind: PersistentVolume
metadata:
  name: filestore-pv
spec:
  capacity:
    storage: 1Ti
  volumeMode: Filesystem
  accessModes:
    - ReadWriteMany
  persistentVolumeReclaimPolicy: Retain
  storageClassName: filestore-csi
  csi:
    driver: filestore.csi.storage.gke.io
    volumeHandle: "projects/$PROJECT_ID/locations/$REGION/instances/my-filestore"
    volumeAttributes:
      volumeName: "myvolume"
EOF
```
